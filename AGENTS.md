Background  
Breba Consulting offers websites and landing pages that clients can update without ongoing developer/consultant support, emphasizing control, no vendor lock-in, and optional use of agents/tools for changes.

Website Description  
Marketing/lead-generation site for consulting services around automated, easily updatable websites and workflows for modern, growth-oriented teams. Focus on automation, automated workflows, conversions, and analytics setup. Messaging centered on customer problems and outcomes. Tools and agents positioned as optional enablers, with a core AI agent focused on content updates. Messaging avoids implying long timelines; landing pages may launch within hours, more complex websites in two or more weeks. Future-facing automation positioning with a concise, punchy automation tagline.

Visual Preferences  
Dark theme. Modern, polished, “productized SaaS/agency” feel. Clean typography with strong hierarchy. Blue and purple accents. Preference for professionally styled numbered elements instead of traditional icons; avoids small or badge-like numbers. Hero heading gradient text at 135° (top line blue-to-purple left to right, bottom line purple-to-blue left to right). Hero heading font weight 600. All section headings and subheadings centered. Navbar primary CTA visually restrained on mobile. No social media icons in footer. Logo uses a single continuous left-to-right gradient across the full “Breba Consulting” text (90°, #8B5CF6 to #60A5FA). Logo hover state avoids unexpected or inconsistent color changes and stays aligned with the hero heading color scheme.

Technical Requirements  
Single-page marketing site. Single HTML file containing markup, styles, and scripts. Contact form submits to an external API endpoint.

Design Constraints / Decisions  
Hero main heading max font size 3.6rem. Hero heading split across two fixed lines (“Your Website” first, “Our Problem” second) with no dashes. Hero heading uses sequential animation with slight delay: top line pops in place, bottom line slides in from the left. Hero section layout fully centered without a separate right-side element; hero content fixed. Hero checklist uses three items that cycle one at a time in a continuous loop, each visible ~5 seconds. “Why Breba” card 2 focused on automation and automated workflows rather than AI; agents only optionally referenced. Single testimonial attributed to Maya Bichikashvili. Mid-tier “Launch Essentials” package treated as the most popular option and explicitly labeled as such. Pricing shows an original price alongside a lower promotional price. Top-tier primary price given equal visual emphasis to mid-tier primary price and not struck through. Top-tier custom option labeled “Custom Pricing” and styled as a normal price. Buttons within comparable cards aligned at the bottom so they sit on the same vertical level. Navbar contains only a “Pricing” link and a primary “Let’s Go” CTA. On desktop, navbar links and CTA initially hidden and become visible with an animated coming-into-focus style transition as soon as the navbar reaches the hero CTA buttons’ vertical position, with a special animated reveal for “Pricing” and “Let’s Go.” On mobile, navbar shows “Pricing” and “Let’s Go” directly with no hamburger menu.
